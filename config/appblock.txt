# StraitJacket application blocklist — one entry per line.
# Each entry may be a bundle id (com.google.Chrome), an .app path
# (/Applications/Google Chrome.app), or a bare executable name (steam).
# Blocked apps owned by the child are killed within ~2s (poll-and-kill), and a
# deny-execute ACL is applied where possible (non-SIP apps).
# Edit with: sudo parentctl block-app <entry> / unblock-app <entry>

# ============================================================
# Web browsers — BLOCK EVERYTHING EXCEPT FIREFOX
# ------------------------------------------------------------
# WHY: per-subdomain DNS/hosts blocking is defeated by HTTP/2 connection
# coalescing. Safari (WebKit) and Chrome/Edge/Brave/etc. (Chromium) reuse an
# existing TLS connection to an ALLOWED Google host (e.g. play.google.com) to
# fetch a BLOCKED one (e.g. news.google.com) over the same socket — Google's
# wildcard cert covers both, so no DNS lookup happens and the sinkhole/hosts
# layers are never consulted. Firefox re-checks DNS and honors the block, so we
# standardize the child on Firefox and block the coalescing browsers.
# See docs/browser-policy.md for the full rationale.
#
# NOTE: org.mozilla.firefox is intentionally NOT listed (it is the allowed
# browser). Do not add a bare "firefox" exec-name rule — Tor Browser and other
# Firefox forks also run an executable named "firefox"; they are blocked here by
# their own bundle ids instead.
# ============================================================
